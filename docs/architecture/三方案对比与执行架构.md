## 二、最优路径：混血方案

不三选一。三个阶段各取三个方案最有用的部分：

```
阶段 0（现在 → 2 周）
  取 A 的垂直场景 + 一人开工
  + 取 B 的云 API 聚合层（轻量版）
  + 服务自己的内容管线
  → 产出：算力通 CLI + 本地 GPU 调度脚本

阶段 1（2 周 → 3 个月）
  取 B 的多云比价引擎 + Agent 体系
  + 取 C 的设备心跳机制（简化版）
  + 接入 2-3 个创作者设备
  → 产出：极简 Web 控制台 + 设备 Agent

阶段 2（3 个月 → 12 个月）
  取 B 的代理返佣模式 + 融资逻辑
  + 取 C 的设备池规模化思路
  → 产出：PMF 验证数据，种子轮就绪
```

## 三、技术栈

| 层级 | 技术 | 理由 |
|------|------|------|
| 后端框架 | Python 3.11+ / FastAPI | 异步支持好，云 SDK 生态完善，CC/Codex 生成质量高 |
| 前端 | React 18 + TypeScript + Tailwind CSS | 社区最成熟，AI 生成代码质量最高 |
| 数据库（MVP） | SQLite | 零配置，单文件，快速启动 |
| 数据库（生产） | PostgreSQL + Redis | 关系型 + 缓存，GPU 实例状态热数据 |
| 消息队列 | Celery + Redis | 异步任务（GPU 实例创建/释放/监控） |
| 容器 | Docker + docker-compose | 设备 Agent 隔离执行，预置镜像打包 |
| 云 SDK | alibabacloud-sdk, huaweicloud-sdk, tencentcloud-sdk | 多云比价核心依赖 |
| CI/CD | GitHub Actions | 自动测试 + 构建 + 部署 |
| 前端部署 | Vercel | 免费档，自动 HTTPS |
| 后端部署 | 阿里云 ECS（2c4g 起步） | 国内访问快，与 GPU 实例同区域低延迟 |

## 五、Agent 编排架构

### Agent 映射表

| Agent 名称 | 类型 | 负责模块 | 核心职责 |
|------------|------|----------|----------|
| `backend-architect` | Plan | 全局架构 | 系统设计、API 设计、数据库 Schema |
| `api-developer` | general-purpose | 所有后端模块 | FastAPI 路由、云 SDK 适配、计费引擎 |
| `frontend-developer` | general-purpose | 用户控制台 | React 组件、状态管理、API 对接 |
| `agent-developer` | general-purpose | 设备 Agent | Python daemon、资源监控、心跳协议 |
| `devops-engineer` | general-purpose | 基础设施 | Docker、docker-compose、GitHub Actions |
| `code-reviewer` | 审查类 | 所有模块 | 代码审查、安全扫描、性能检查 |

### Agent 协作链路

```
backend-architect（出设计）
    ↓
┌───────────────────┬───────────────────────┬──────────────────┐
▼                   ▼                       ▼                  ▼
api-developer    agent-developer    frontend-developer    devops-engineer
（后端 API）      （设备客户端）       （Web 控制台）         （部署管线）
    │                   │                       │                  │
    └───────────────────┴───────────────────────┴──────────────────┘
                        │
                        ▼
                 code-reviewer（审查通过 → 合并）
```

### 并行策略

- **阶段 0/1**：`api-developer` + `agent-developer` 并行开发（后端 API 和设备 Agent 无依赖）
- **阶段 1 后期**：上述完成后，`frontend-developer` 启动（依赖 API 接口稳定）
- **全程**：`devops-engineer` 独立并行（Docker 配置 + CI/CD 不依赖业务代码）

## 七、阶段 0 执行步骤

按优先级排列，括号内为负责 Agent：

```
□ 1. 多云比价 CLI 脚本                     [api-developer]
     输入：GPU 型号（4090/A100/H100）
     输出：阿里云/华为云/腾讯云实时价格对比表
     工期：1 天

□ 2. SQLite 数据库 Schema                  [backend-architect → api-developer]
     表：users, gpu_instances, price_snapshots, billing_records
     工期：半天

□ 3. FastAPI 骨架                           [api-developer]
     GET  /api/v1/prices?gpu_type=A100
     POST /api/v1/instances  (创建 GPU 实例)
     GET  /api/v1/instances  (列表)
     DELETE /api/v1/instances/:id
     工期：1 天

□ 4. Docker 预置镜像（PyTorch + Jupyter）    [devops-engineer]
     工期：半天

□ 5. 设备 Agent 最小原型                     [agent-developer]
     检测本机 GPU 闲时 → 执行 Docker 任务 → 上报心跳
     工期：1 天

□ 6. React 控制台骨架                        [frontend-developer]
     比价页 + 实例列表 + 余额显示
     工期：1 天

□ 7. docker-compose 一键启动                 [devops-engineer]
     工期：半天
```

**总计：约 5 个工作日（并行后压缩为 2-3 天）**
